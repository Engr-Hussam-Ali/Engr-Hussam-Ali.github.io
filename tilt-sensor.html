Tilt Sensor PCB Assembly FEA & Optimization Study

Maximizing IoT tilt sensor signal reliability through multiphysics FEM analysis, systematic mesh convergence, and surrogate-model-driven optimal screw mounting position design.

Abaqus/Standard 3DEXPERIENCE R2018x Mesh Convergence DOE Surrogate Modeling Thermo-Structural
01 — Problem Statement

Problem Definition

Safety IoT tilt sensors deployed for structural health monitoring of aging buildings suffer from signal reliability degradation caused by environmental noise factors—principally temperature and humidity—that induce parasitic deformation in the sensor package surrounding the MEMS element.

The sensor output signal contains noise contributions from both the MEMS element and the surrounding package structure; minimizing the package contribution through enclosure structural optimization is the primary objective, as the package contribution dominates over the element contribution.

y(x, n) = yt(x) + ys(n) + ye(n)

Where: yt = tilt signal of interest, ys = MEMS element noise, ye = package structural noise (optimization target)

Achieving this requires a numerically trustworthy FEM baseline—validated through rigorous mesh convergence, geometry-based constraint definitions, and tie-surface partitioning—before any DOE-driven parametric study or surrogate-model optimization can yield meaningful design recommendations.

03 — Methodology

Five-Phase Analysis Strategy

Phase 01
Multiphysics FE Model Construction

Thermo-structural coupled analysis in Abaqus/Standard. Assembly of 7 component types with 20 tie constraints. Shell + solid hybrid element formulation with surface-to-surface contact discretization.

Phase 02
Systematic Mesh Convergence Study

Sequential refinement of large PCB (2.0 → 0.125 mm) and small PCB (2.0 → 0.125 mm) mesh seeds, monitoring UR1/UR2/UR3 at MEMS center node (Node 41, MEMS3-1). Convergence threshold: <1–2% change between successive sizes.

Phase 03
Root-Cause Diagnosis & 5 Iterative Fixes

Diagnosed and resolved 5 systematic issues: (F1) node-based → element-based CYL surfaces, (F2) node-based → geometry-based MEMS sets, (F3) cylinder pad partitioning, (F4) ABS housing footprint partitioning with buffer zones, and (F5) large PCB 0.1° orientation correction—the final breakthrough that eliminated all tie reversions and singularities.

Phase 04
DOE Parametric Study (100 FEA Models)

Latin Hypercube Sampling over screw mounting positions (hole1, hole2, hole3 XY coordinates). Each DOE point = full Abaqus solve extracting MEMS rotational deformation (UR1, UR2, UR3).

Phase 05
Surrogate Modeling & Optimization

Gaussian Process surrogate from DOE dataset. Objective: maximize tilt sensitivity (yt) while minimizing environmental noise response (ye). Adaptive sampling to refine high-uncertainty regions. Cross-validation for model accuracy.

05 — Diagnostic Log

Root-Cause Fix History

Each fix resolved a systematic source of non-convergence or non-physical results. Pre-fix results cannot be trusted as baselines. The orientation fix (F5) was the final breakthrough.

BEFORE

  • 5 tie reversions per run
  • 9 numerical singularities
  • 2 negative eigenvalues
  • 176 "WILL NOT BE TIED" warnings
  • Non-convergent at 0.125 mm (~3.4% oscillation)

AFTER

  • 0 tie reversions
  • 0 numerical singularities
  • 0 negative eigenvalues
  • 143 "WILL NOT BE TIED" warnings (persistent housing)
  • Textbook convergence — UR1: 0.15%, UR2: 0.07%
F1
Node-Based → Element-Based Master Surfaces

Replaced node-based CYL1/2/3 master surfaces with element-based surfaces using surface-to-surface discretization. Node IDs reassign silently after every remesh.

F2
Node-Based → Geometry-Based MEMS Sets

Replaced node-based MEMS sets with geometry-based element surfaces. Eliminates constraint failures caused by node ID reassignment on remesh.

F3
Cylinder Pad Partitioning

Geometrically partitioned the large PCB at cylinder footprints, creating dedicated pad surfaces. Ensures identical element faces participate in ties at every mesh size.

F4
ABS Housing Footprint Partitioning

Geometric partitioning of the large PCB at all 3 ABS housing (pl, pl2, pl3) contact footprints with buffer zones. Ensures consistent element face participation in pcb_abs_1/2/3 ties across mesh sizes.

F5
Large PCB Orientation Correction (0.1° Tilt)

Discovered the large PCB had a 0.1° tilt—normal was (-1.745E-03, 0, 999.998E-03) instead of (0, 0, 1). Components below had pure Z normals, creating a wedge-shaped gap at every tie interface. Rotated PCB and all components above it to correct orientation. Result: 0 tie reversions, 0 singularities, 0 negative eigenvalues. Tie failures dropped from 176 to 143. This was the final fix that enabled textbook convergence at 0.125 mm.

07 — Engineering Insights

Key Learnings & Principles

1

Node-Based Sets Are Fragile — Node IDs reassign silently after every remesh, causing constraint failures and non-physical results. All sets must be geometry-based or element-based. This single root cause invalidated early convergence baselines entirely.

2

Read the .dat, Not Just the .msg — The breakthrough that unlocked diagnosis came from reading the .dat file for preprocessing warnings—not the .msg file. 320+ hidden warnings were identified that the .msg completely omitted.

3

Sub-Degree Geometry Misalignment Kills Convergence — A 0.1° tilt in the large PCB—invisible to the eye—created a wedge-shaped gap at every tie interface, causing persistent tie reversions and non-convergent results.

4

Partitioning Is Essential for Tie Stability — Without geometric partitioning at contact footprints (3 screw pads + 3 ABS footprints on large PCB), different element faces participate in tie constraints at each mesh size.

5

Validate Constraint Sensitivity Last — After achieving convergence, constraint idealization changes affect results by only 0.3–1.9%. This test is only meaningful on a healthy model.
